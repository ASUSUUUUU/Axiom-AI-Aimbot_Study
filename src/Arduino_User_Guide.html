Arduino Leonardo Setup Guide

Hardware-level mouse control for Axiom

Using an Arduino Leonardo allows Axiom to simulate mouse movements at the hardware level. This makes the aim assistance extremely stealthy and indistinguishable from real mouse inputs to most software.

Requirements

  • Hardware: Arduino Leonardo (ATmega32U4) or compatible clone.
  • Cable: Micro USB Data Cable (ensure it supports data transfer, not just charging).
  • Software: Arduino IDE (v1.8.x or v2.x).

1 Install Arduino IDE

Download and install the Arduino IDE from the official website:

https://www.arduino.cc/en/software

2 Upload the Sketch (win_utils/arduino_spoof.ino)

1. Connect your Arduino Leonardo to your PC via USB.

2. Open the file src/win_utils/arduino_spoof.ino with Arduino IDE.

3. In Arduino IDE, go to Tools > Board and select "Arduino Leonardo".

4. Go to Tools > Port and select the COM port labeled "Arduino Leonardo".

5. Click the Upload button (Arrow icon) in the top left.

6. Wait for the "Done uploading" message.

3 Device Spoofing (Optional but Recommended)

Why Spoof? By default, the device shows up as "Arduino Leonardo" in Device Manager. Anti-cheat systems may flag this. Spoofing changes the Vendor ID (VID) and Product ID (PID) to match a generic mouse (e.g., Logitech), making it look like a regular peripheral.

1. Close Arduino IDE.

2. In the Axiom app, go to the Aim Assist tab.

3. Select "Arduino Leonardo" as the Mouse Move Method.

4. Click the purple "One-click Device Spoofing" button.

5. Confirm the prompt to modify your Arduino boards.txt file.

6. Once successful, Restart Arduino IDE.

7. Re-upload the sketch to your Arduino (Repeat Step 2). The Arduino acts as a new device now.

4 Configure Axiom

1. Open Axiom.

2. Go to the Aim Assist tab.

3. Set Mouse Move Method to "Arduino Leonardo".

4. In the Arduino COM Port dropdown, select the port corresponding to your Arduino.

5. If connected successfully, you will see a green âœ“ Connected status.

6. You can use the "Test Heart Move" button to verify it's working.

Troubleshooting

Connection Failed / No COM Port Found

  • Click the "Refresh" button next to the COM port dropdown.
  • Ensure the USB cable is a data cable, not just a charging cable.
  • Check Windows Device Manager under "Ports (COM & LPT)" to see if the device appears.
  • If you spoofed the device, it might show up as a "USB Input Device" or "Logitech Mouse" instead of Arduino, but the COM port should still be visible.

Mouse Moves Erratically

  • Ensure the baud rate in the Arduino sketch (115200) matches the Python script (default is correct).
  • Try a different USB port on your computer.
Important: If you use the "One-click Device Spoofing", remember that this modifies your local Arduino IDE installation files. If you update the Arduino IDE, you may need to run the spoofing tool again.
